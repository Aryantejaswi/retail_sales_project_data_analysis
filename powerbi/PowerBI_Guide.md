# Power BI Dashboard — Setup Guide

Power BI files (`.pbix`) are binary and can't be generated by code, so this
guide gets you from the cleaned data to a finished dashboard in Power BI
Desktop. Budget ~30–45 minutes.

## 1. Connect to the data

You have two options — pick whichever you want to show off:

**Option A — Connect directly to SQLite (shows a real DB connection):**
1. Install the free "SQLite ODBC Driver" (e.g. from `ch-werner.de/sqliteodbc`).
2. In Power BI Desktop: `Get Data` → `ODBC` → select the SQLite DSN pointing
   at `data/sales.db` → load the `sales` table.

**Option B — Simplest (works everywhere, no driver install):**
1. `Get Data` → `Text/CSV` → select `data/clean_sales.csv`.
2. Click `Transform Data` to open Power Query, confirm data types
   (`order_date` = Date, numeric columns = Decimal/Whole Number).

## 2. Build the data model

This project is a single flat table, so no relationships are needed. If you
want to demonstrate modeling skills, split it into a star schema instead:

- `Dim_Date` (order_date, year, month, quarter)
- `Dim_Customer` (customer_id, customer_segment)
- `Dim_Product` (product, category)
- `Fact_Sales` (order_id, date key, customer key, product key, unit_price,
  quantity, discount_amount, net_sales, payment_method, region, city)

Power Query's `Reference` + `Remove Duplicates` on each dimension is the
fastest way to build this from the flat CSV.

## 3. DAX measures to add

Create a new Measures table (`Modeling` → `New Table` → name it `_Measures`),
then add these:

```DAX
Total Revenue = SUM(sales[net_sales])

Total Orders = DISTINCTCOUNT(sales[order_id])

Total Customers = DISTINCTCOUNT(sales[customer_id])

Avg Order Value = DIVIDE([Total Revenue], [Total Orders])

Total Discount = SUM(sales[discount_amount])

Discount Rate % = DIVIDE([Total Discount], SUM(sales[gross_sales]))

Revenue per Customer = DIVIDE([Total Revenue], [Total Customers])

-- Time intelligence (requires a proper Date table marked as such)
Revenue LM =
CALCULATE([Total Revenue], DATEADD('Dim_Date'[order_date], -1, MONTH))

Revenue MoM % =
DIVIDE([Total Revenue] - [Revenue LM], [Revenue LM])

Revenue YTD =
TOTALYTD([Total Revenue], 'Dim_Date'[order_date])
```

## 4. Suggested pages & visuals

**Page 1 — Executive Overview**
- KPI cards: Total Revenue, Total Orders, Avg Order Value, Total Customers
- Line chart: Revenue by Month (with MoM % as a secondary line or tooltip)
- Bar chart: Revenue by Region
- Donut chart: Revenue by Customer Segment
- Slicers: Year, Region, Category

**Page 2 — Product Performance**
- Table/matrix: Category → Product, sorted by revenue, with units sold and
  discount rate
- Top 10 Products bar chart
- Scatter plot: Discount Rate % vs. Net Revenue per category (spot categories
  being over-discounted)

**Page 3 — Customer Insights**
- Table: Top 10 customers by lifetime value
- Bar chart: Revenue per Customer by segment
- Map visual: Revenue by City (use `city` + `region` for geocoding)

## 5. Design polish (for portfolio screenshots)

- Use a consistent color theme (`View` → `Themes`) — one accent color per
  category/region across all pages.
- Add a title bar and page navigation buttons.
- Turn on tooltips with a small drill-through page per product/customer.
- Export each page as an image (`File` → `Export` → `PDF`, then screenshot
  pages) to embed in your README/portfolio site.

## 6. What to write in your case study

When presenting this project (LinkedIn, GitHub README, portfolio site),
structure the write-up around:
1. **Business question** — e.g. "Which regions/categories are underperforming
   and why?"
2. **Data pipeline** — Python (synthetic data + cleaning) → SQL (analysis) →
   Power BI (visualization)
3. **Key findings** — pull 2–3 concrete numbers from your queries/charts
4. **Recommendation** — a decision the business could make from the dashboard
