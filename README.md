# Retail Sales Performance Analysis — End-to-End Data Analyst Project

An end-to-end portfolio project demonstrating a full analytics workflow:
**Python (data generation + ETL + EDA) → SQL (SQLite, analytical queries) →
Power BI (interactive dashboard)**.

## Project Overview

**Business question:** How is revenue trending across regions, categories,
and customer segments, and where should the business focus to grow?

**Pipeline:**

```
generate_data.py  →  raw_sales.csv  (messy, realistic transactional data)
        │
        ▼
   etl_load.py     →  clean_sales.csv + sales.db (SQLite)
        │
        ├──────────────► analysis_queries.sql  (SQL analysis, 10 queries)
        │
        ▼
 eda_analysis.py    →  charts in /outputs (Python EDA)
        │
        ▼
   Power BI          →  interactive dashboard (see powerbi/PowerBI_Guide.md)
```

## Folder Structure

```
retail_sales_portfolio/
├── data/
│   ├── raw_sales.csv        # generated, intentionally messy
│   ├── clean_sales.csv      # output of ETL
│   └── sales.db             # SQLite database (table: sales)
├── python/
│   ├── generate_data.py     # creates the synthetic raw dataset
│   ├── etl_load.py          # cleans data, loads into SQLite
│   └── eda_analysis.py      # SQL-backed EDA + chart generation
├── sql/
│   ├── schema.sql            # table definition
│   └── analysis_queries.sql  # 10 analyst-style SQL queries
├── powerbi/
│   └── PowerBI_Guide.md      # step-by-step dashboard build guide + DAX
└── outputs/                  # charts generated by eda_analysis.py
```

## How to Run

```bash
# 1. Install dependencies
pip install pandas numpy matplotlib seaborn

# 2. Generate the raw (messy) dataset
cd python
python generate_data.py

# 3. Clean it and load into SQLite
python etl_load.py

# 4. Run EDA (produces charts in ../outputs/)
python eda_analysis.py

# 5. Explore SQL queries directly (optional)
sqlite3 ../data/sales.db < ../sql/analysis_queries.sql

# 6. Build the Power BI dashboard
# Follow ../powerbi/PowerBI_Guide.md, connecting to data/clean_sales.csv
# or data/sales.db
```
